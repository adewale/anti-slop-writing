# Failure: Weak conclusion

## Original

```txt
A benchmark is stronger when you can inspect the run that produced it.
```

## Why it failed

The sentence is true, but it could end many essays about evaluation observability. It drops the concrete carrier that made the argument memorable.

## Mechanism

A thesis-only ending loses the object that carries the claim.

## Better rewrite

```txt
Because the pelican project is small enough to inspect and strange enough to remember, it works as a compact carrier for the larger claim: a benchmark is stronger when you can inspect the run that produced it.
```

## Rule added or changed

A conclusion should return to the concrete carrier, name why it matters, and then state what transfers.
