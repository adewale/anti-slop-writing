## Method

Four fresh subagent contexts were spawned in parallel. Each subagent read `skills/anti-slop-writing/SKILL.md` and `skills/anti-slop-writing/references/anti-slop-writing-doctrine.md` only — no prior conversation context — and ran a subset of the new evals end-to-end:

- Subagent 1: four new critique evals in `evals/evals.json`
- Subagent 2: three new adversarial evals in `evals/adversarial.json`
- Subagent 3: three new rewrite evals in `evals/rewrite-evals.json`
- Subagent 4: one meta-eval in `evals/meta-evals.json` + two new trigger queries

Each subagent produced a real response for each prompt in the skill's prescribed format, then self-judged every assertion with quoted evidence. The instruction was explicit: be brutally honest, default to FAIL on ambiguous matches.

## What this measures and what it does not

What the smoke run does measure:

- The doctrine, when loaded into a fresh agent context, produces text that satisfies each new assertion when asked to critique or rewrite the prompt.
- The adversarial counterparts hold: with the doctrine loaded, the agent did not over-flag earned `serves`, earned `Whether X or Y`, or earned em-dash usage. All three adversarial verdicts were `keep`.
- The skill description scopes the two new trigger queries correctly.

What it does not measure:

- **Self-judgment bias.** The same agent that produced the response also graded it against the assertions. An independent grader with no doctrine loaded would be a stricter test. The high pass rate partly reflects vocabulary alignment between doctrine and assertions.
- **Trigger-rate behavior in real clients.** The trigger-query evaluation was a meta-judgment from the skill description, not an observed routing decision in Pi, Claude Code, Codex, or OpenCode. A real trigger run with `runs_per_query: 3` against each client is still outstanding.
- **Long-form prose at scale.** The evals are sentence-to-paragraph. Performance on a full 2,000-word article (where multiple tells co-occur and rewrite quality compounds) is not directly tested by these cases.
- **Drift over model generations.** The smoke run reflects current model behavior with current doctrine. The `word-list-drift` meta-eval is the only check that addresses this, and it tests reasoning *about* drift, not actual drift recovery.

## Assessment

The eval contract added in commit `47d242d` is satisfied by the doctrine update in commit `18a7c8e`, as measured by self-judged fresh-context smoke runs: 36/36. The four limitations above are the next measurement steps in order of value: independent grader > observed trigger rates > long-form runs > drift recovery.
