# Baseline — 2026-05-29

The first end-to-end scored run of the current skill against every tune and holdout case, using the sub-agent judge protocol in `docs/judge-protocol.md` and the runner in `scripts/run_evals.py`. This is the comparison point future rounds `join` against.

Artifacts live under `evals/results/baseline-2026-05-29/`:

```
outputs/<suite>/<id>.md       38 skill-applied outputs
judgments/<suite>.jsonl       per-assertion judgments (4 suites)
judgments/trigger.jsonl       26 routing decisions
scores.jsonl                  per-case scores (the recorded baseline)
```

## How it was produced

- **Apply**: four sub-agents (one per suite) read `SKILL.md` and applied it to each case prompt, blind to the assertions, writing one output file per case.
- **Judge**: four separate sub-agents read those outputs and the assertions, judged each assertion with quoted evidence, and wrote judgment JSONL. Graded dimensions were scored where the case defines them.
- **Grade**: `python3 scripts/run_evals.py grade judgments/{evals,adversarial,rewrite,meta}.jsonl --out scores.jsonl`.
- **Trigger**: one routing sub-agent decided activation per query from the skill description alone, then compared to the label.

## Graded-dimension results (the discriminating signal)

The binary assertions are at ceiling. The orthogonal `graded_dimensions` are not:

| Case | Split | Dimensions |
|---|---|---|
| `durable-execution-mechanism` | tune | specificity=5, **length-control=1** |
| `fake-precision-rewrite-finance` | holdout | provenance=5, **length-control=1** |
| `pelican-process-relation` | tune | relation-clarity=5 |
| `product-tour-rewrite-developer-tools` | holdout | relation-clarity=5 |

The two `length-control=1` scores are the most interesting result, but not for the reason they first appear. Reading the actual outputs, neither rewrite is padded:

- `durable-execution-mechanism` rewrote a 10-word vague sentence into one tight 25-word sentence that names the retry-from-checkpoint mechanism. It is longer than the input *because adding the mechanism is the skill's job*.
- `fake-precision-rewrite-finance` is long because the case carries a `dynamic_rubric` that asks the agent to draft per-instance criteria, and because the case legitimately needs two conditional rewrites (provenance-exists vs not).

So the finding is a **rubric-calibration bug, not a skill regression**. The `length-control` rubric as written — "5 = same or shorter word count than input" — penalizes the skill for doing exactly what it should: replacing a vague short sentence with a concrete, necessarily longer one. The real anti-slop length concern is padding (filler clauses, redundant hedging), not raw length-versus-input.

That is still item #4 (orthogonal axes) and item #13 (continuous metrics) earning their place: a single binary pass/fail would have shown 38/38 and surfaced nothing. The graded axis surfaced a real problem on the first run — it just turned out to be a problem with the measurement, which is precisely what `evals/meta-evals.json` → `judge-drift-evidence` exists to catch.

## Assessment

- The binary suite is **saturated**: 115/115 and 26/26. Per our own `evals/meta-evals.json` → `ceiling-effect-detection`, that is a coverage warning, not a finish line.
- The discriminating signal now lives in (a) the graded dimensions, which on this run exposed a rubric-calibration bug rather than a skill defect, and (b) the discourse-layer cases that do not yet exist (see `Lessons_learned.md` → "Doctrine reaches the surface layer" and `TODO.md`).
- **Known limitation**: apply and judge sub-agents are the same model family (Claude). Self-preference is a documented risk (`Lessons_learned.md` → "Rewrite-eval grading inherited known judge biases"). A cross-family judge is not reachable in this harness; treat the binary scores as a coverage signal, not a calibrated number. The `length-control=1` findings are more trustworthy precisely because they cut against any self-preference — the judge flagged the skill's own output.

## Next round

1. **Fix the `length-control` rubric** (no new examples needed, do this first). Redefine it to measure padding — filler clauses, redundant hedging, restated points — rather than word count against the input. A mechanism-adding rewrite that is longer than a vague original should score well; a rewrite that pads should not. This is an eval-calibration change, gated like any other: re-judge the affected rewrite cases and confirm the scores move for the right reason.
2. Add discourse-layer failure examples (blocked on a human — `TODO.md`). Once they land, the doctrine work that needs them can begin.

Note what did *not* happen here: the baseline did not justify a doctrine change. The binary suite is at ceiling and the one graded-axis failure was a measurement bug. Editing `SKILL.md` on this evidence would be hillclimbing on noise — exactly what the held-out gate and `score_delta.py` exist to prevent.
